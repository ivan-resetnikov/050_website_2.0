Selling the signals feature

Signals is a Godot-native implementation of the observer-pattern.
Signals define a relationship of event → consequence(s) while maintaining low coupling of your code! (very cool)

People often use signals like this,
but Godot does not have a way to do it natively

Imagine your designer walks up to you, hands you the script and goes like:

Shortcommings of the original implementation

  • One object of SignalGroup holds its own array of signals that it can wait for, following some strategy. There is an off-change that a SignalGroup will be reused without waiting for the old batch to finish. That will break the SignalGroup...

  • So, I created SignalGroup factories. all and any to create and await for the awaitables without worrying about state corruption, and thinking about variable names.

  • I made use of variadic arguments added to Godot in #1034, to prevent a crash when awaiting for a signal that passes a parameter.

  • I added support for couroutines too!
